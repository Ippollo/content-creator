# Blog Article Checklist

Run each check against the drafted article. The article must pass ALL items before proceeding to content-reviewer scoring.

## Mechanical Checks

These are binary — the article either passes or fails. No judgment call needed.

- [ ] **No em dashes (—)**: Search for `—`. If found, replace with a period, comma, or restructure the sentence.
- [ ] **Word count 1,500–2,500**: Count the full article (excluding frontmatter and FAQ). Flag if outside range.
- [ ] **H1 is the title only**: Exactly one H1 in the document.
- [ ] **No skipped heading levels**: H1 → H2 → H3. No H1 → H3 jumps.
- [ ] **Section length under 300 words**: Each H2 section stays under 300 words. If any section exceeds this, break into H3 subsections or split into two H2s.
- [ ] **FAQ placement**: If an FAQ section exists, it appears after the conclusion and before sources. Never mid-article.
- [ ] **Meta description 145–155 characters**: Check the frontmatter `meta_description` field.
- [ ] **Target keyword in title**: The frontmatter `target_keyword` appears in the article title.
- [ ] **Target keyword in first 100 words**: The primary keyword appears naturally within the first 100 words of the article body.
- [ ] **Target keyword in 2+ H2s**: The primary keyword or a close variant appears in at least two H2 headings.
- [ ] **All stats have named sources**: Every number or statistic traces to a named source (e.g., "Deloitte's 2026 State of AI report"). No unattributed stats.

## Voice Checks

These require reading against the voice profile. Flag any violation.

- [ ] **No corporate-speak**: Scan for phrases like "In today's fast-paced world", "It's important to note", "leverage", "synergies", "actionable insights", "innovative approach". If found, rewrite in plain language.
- [ ] **No AI tells**: The article should not sound like it was generated by an AI. Check for overly smooth prose, hedging language, perfect parallel structure across all points, and "on one hand / on the other hand" framing.
- [ ] **No over-explanation**: If a line makes a point and the next line re-explains it in different words, cut the second line. Trust the reader.
- [ ] **Voice consistency**: Read the intro and the last body section back-to-back. Do they sound like the same person wrote them? Flag sections where the tone shifts from conversational operator to formal consultant.
- [ ] **No blacklist patterns**: Scan for "Not X, but Y" correlatives, hedges ("actually," "maybe," "just," "simply"), throat-clearing ("It's worth noting that..."), false profundity ("At the end of the day..."), and generic transitions ("That said," "Moreover," "Furthermore").

## Structural Checks

These verify article architecture. The most important checks for long-form quality.

- [ ] **Thesis clear by paragraph 3**: A reader should know what the article argues within the first three paragraphs. If the intro is scene-setting without a thesis, restructure.
- [ ] **Concrete hook**: Opens with a specific number, scenario, or bold claim — not an abstract question or category label. The opening line should stop someone from clicking away.
- [ ] **Argument progression**: Run the reorder test — could any two H2 sections swap positions without breaking the article's logic? If yes, sections are stacked in parallel rather than building an argument. Restructure so each section builds on the previous one's conclusion.
- [ ] **Section transitions connect**: Check the last sentence of each H2 section and the first sentence of the next. Is there a logical bridge, or do sections just start on a new topic? Transitions should feel like the next question the reader naturally asks.
- [ ] **Evidence density**: Count distinct named sources across the article. Minimum 3 for a 1,500+ word piece. If fewer, flag as thin sourcing.
- [ ] **H2 headers hook**: Each H2 should make the reader want to read the section. Generic descriptive headers ("AI Tool Distribution," "Implementation Approaches") fail this check. Headers should create curiosity or promise value.
- [ ] **Single core argument**: The article serves one thesis. If a section introduces a second, unrelated argument thread, flag it as a second idea — cut or rework.
- [ ] **Closing reframes, doesn't summarize**: The conclusion must extend or reframe the thesis, not restate it. A one-line question the reader has to answer for themselves is the gold standard. Any paragraph that starts with "In conclusion" or "To summarize" fails.
- [ ] **Cascading lists**: If the article uses bullet or numbered lists, items should create a visual slope — progressively shorter or longer. Rewrite items to achieve the cascade if logical order prevents reordering.
