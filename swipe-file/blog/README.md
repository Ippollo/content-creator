# Blog Swipe File

Curated examples of long-form writing that demonstrate effective structure, argumentation, and voice. The `blog-writer` skill references this directory for patterns before writing.

## Index

### Own Articles (published content for pattern consistency)

- **01** — Harness Engineering Operator's Guide — hook patterns, section rhythm, H2 structure
- **02** — AI Memory Architecture — technical explainer structure
- **03** — Enterprise AI Strategy — evidence layering, research integration

### External Writers (studied for specific techniques)

- **04** — Cal Newport: "Knowledge Workers are Bad at Working" — **Evidence Ladder**, progressive argumentation, research integration at blog length
- **05** — Will Larson: "Solving the Engineering Strategy Crisis" — **Constraint-First Framing**, trade-off reasoning, real company case studies
- **06** — Alex Hormozi: Value Equation ($100M Offers) — **Compression**, Operational Proof Chain, named framework deployment
- **07** — Robert Greene: The Perfect Courtier (48 Laws of Power) — **Narrative architecture**, progressive case-building, the accumulation pattern

## How to Use

Each external entry has a **Technique Map** section that breaks down specific moves the writer makes, annotated with:

1. **What to learn** — the transferable technique
2. **How Chris would adapt** — how the pattern maps to our voice and content type
3. **Anti-patterns to avoid** — what NOT to adopt from this writer

When writing a new article, scan the technique maps for the specific challenge you're facing:

| Writing Challenge | Study Entry |
|---|---|
| Sections feel parallel / reorderable | 04 (Newport) — Evidence Ladder |
| Claims feel ungrounded | 06 (Hormozi) — Dollar-anchored proof |
| Solutions appear without context | 05 (Larson) — Constraint-first case studies |
| Opening feels generic | 07 (Greene) — Story-before-thesis |
| Evidence feels thin | 07 (Greene) — Accumulation pattern |
| Transitions feel forced | 05 (Larson) — Negative case as proof |

## File Format

Each entry includes:

- **Why This Is Here** — one-line purpose
- **Technique Map** — annotated breakdown of specific moves
- **Anti-Patterns to Avoid** — what doesn't translate to our voice
- **Content** — the article text (own articles) or representative passages (external)

Learn patterns — never copy content.
